4. การเชื่อมโยงงานกับภาพรวม

ถ้า Frontend ถูกออกแบบและพัฒนาได้ดี จะช่วยให้ผู้ใช้งานเข้าใจระบบได้ง่าย ใช้งานสะดวก เป็นตัวกลางในการเชื่อมต่อระหว่างผู้ใช้กับระบบ Backend และ Blockchain
ข้อควรระวัง: หากมีข้อผิดพลาดในส่วนนี้ เช่น แสดงผลข้อมูลไม่ถูกต้อง หรือเรียกใช้งาน API ผิดพลาด จะส่งผลให้ระบบโดยรวมทำงานไม่สมบูรณ์

5. การวิเคราะห์ปัญหาและการเรียนรู้

ปัญหาที่พบ: มีปัญหาเรื่องระยะเวลาการทำงาน เพราะเครื่องมือบางอย่างไม่เคยทำมาก่อน ทำให้ต้องใช้เวลาในการเรียนรู้สักระยะ

6. แผนงานระยะถัดไป

ตอนนี้ทำตัวแอปพลิเคชันเสร็จเรียบร้อยแล้วแต่ยังมีข้อผิดพลาดในการเขียนโค้ดบางหน้า ในการรันอาจจะมีปัญหาอยู่บ้าง กำลังทำการแก้ไขเพื่อที่จะส่งต่อให้ส่วน Backend รัน API เพื่อให้ใช้งานได้ตามเป้าหมาย
