4. ความสำคัญและภาพรวม

หาก Frontend ถูกออกแบบและพัฒนาได้ดี จะช่วยให้ผู้ใช้งานเข้าใจระบบได้ง่ายดายและสะดวก ถือเป็นตัวกลาง (Bridge) เชื่อมระหว่าง "ผู้ใช้งาน" กับ "Backend & Blockchain"

ข้อควรระวัง: หากมีข้อผิดพลาดในส่วนนี้ (เช่น แสดงข้อมูลผิด, เรียก API พลาด) จะส่งผลให้ระบบทำงานไม่สมบูรณ์และผู้ใช้ขาดความเชื่อมั่น

5. ปัญหาและการเรียนรู้

ปัญหาที่พบ: ระยะเวลา
เนื่องจากมีหลายเทคโนโลยีที่ไม่เคยใช้งานมาก่อน ทำให้ต้องใช้เวลาในการศึกษาและทำความเข้าใจเครื่องมือ (Learning Curve)

การแก้ไข: จัดสรรเวลาในการเรียนรู้และลงมือปฏิบัติจริงไปพร้อมๆ กัน

6. แผนงานระยะถัดไป

เป้าหมายต่อไป:
ส่งต่อแอปพลิเคชันในส่วน Frontend ที่พัฒนาเสร็จแล้วให้ทีม Backend

เพื่อทำการ เชื่อมต่อ API จริง (Integrate APIs) และทดสอบระบบให้พร้อมใช้งานได้ตามแผนที่วางไว้
